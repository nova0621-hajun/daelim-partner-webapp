# 시공관리 웹앱 계정/로그인 1번 묶음 최종 체크리스트

## 1. 기준 시트

- 시트명: `기초데이터`
- 로그인 ID: C열 `이름`
- 임시 비밀번호: F열 공백 또는 `0000`
- 비밀번호상태: I열

## 2. 자동 권한 정책

로그인 또는 계정 목록 조회 시 아래 기준으로 자동 보정한다.

- C열 이름 `최하준`: `master`
- A열 `영업1팀` + C열 이름 `이상민`: `team1_leader`
- A열 `영업2팀` + C열 이름 `김태호`: `team2_leader`
- A열 `영업1팀` 또는 `영업2팀` 일반 인원: `sales`
- A열 공백 또는 영업팀 외 계정: `viewer`

## 3. 자동 활성화 정책

- C열 이름이 있는 계정은 로그인/계정 조회 시 H열 `사용여부`를 `승인`으로 보정한다.
- F열 비밀번호가 공백이면 `0000`을 입력하고 I열 `비밀번호상태`를 `임시`로 보정한다.

## 4. 강제 비밀번호 변경 정책

- F열 비밀번호가 공백이었거나 `0000`이면 로그인 후 강제 비밀번호 변경 화면으로 이동한다.
- 새 비밀번호는 숫자 4자리만 허용한다.
- 새 비밀번호로 `0000`은 사용할 수 없다.
- 변경 완료 후 F열에는 `SHA256:` 해시값이 저장되고 I열은 `정상`으로 변경된다.

## 5. 계정생성 요청 정책

- 로그인 화면의 `계정생성 요청`에서 이름, 소속팀, 휴대전화, 로그인 비밀번호를 입력한다.
- 소속팀은 `영업1팀`, `영업2팀`을 선택하거나 직접 입력할 수 있다.
- 사용자가 설정한 비밀번호는 F열에 `SHA256:` 해시값으로 저장한다.
- 신규 요청 계정은 G열 `viewer`, H열 `대기`, I열 `정상`으로 저장한다.
- master가 승인하면 사용자는 요청 시 설정한 비밀번호 그대로 로그인한다.

## 6. 배포 반영 파일

Apps Script 편집기에 아래 파일을 최신 코드로 반영한 뒤 새 배포 또는 배포 업데이트를 한다.

- `02_Router.gs`
- `09_AdminService.gs`
- `13_PaymentService.gs`
- `14_AdminUserService.gs`

## 7. 배포 전 확인

- `13_PaymentService.gs` 안에 `ADMIN_USER_SHEET_NAME`, `ADMIN_USER_COL` 같은 계정관리 코드가 섞여 있으면 안 된다.
- `14_AdminUserService.gs` 안에는 `DAELIM_ADMIN_ACCOUNT_`로 시작하는 계정 전용 전역 변수만 있어야 한다.
- 파일을 붙여넣을 때 같은 코드가 같은 GS 파일 안에 두 번 들어가지 않도록 확인한다.

## 8. 테스트 순서

1. Apps Script 웹앱 배포 업데이트
2. `...?action=health` 접속 후 `API 정상 연결` 확인
3. 시공관리 웹앱에서 `최하준 / 0000` 로그인
4. 강제 비밀번호 변경 화면 표시 확인
5. 새 비밀번호 설정
6. `기초데이터` F열이 `SHA256:` 값으로 변경되고 I열이 `정상`인지 확인
7. 새 비밀번호로 재로그인
8. 협력사 포털 로그인도 함께 정상 확인
9. 계정생성 요청 후 기초데이터에 `viewer / 대기 / 정상`으로 저장되는지 확인
10. master 승인 후 요청 시 설정한 비밀번호로 로그인되는지 확인

## 9. 1번 묶음 완료 범위

- 시공관리 웹앱 로그인
- 8시간 로그인 유지

- 임시 비밀번호 `0000`
- 강제 비밀번호 변경
- 비밀번호 해시 저장
- master 계정 관리 화면
- 계정생성 요청
- viewer/sales/team/master 권한별 1차 화면 제한
