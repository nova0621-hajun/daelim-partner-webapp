# 시공관리 웹앱 / 협력사 포털 통합 개발 작업 문서

## 기준 방향

- 현장별 비밀번호 방식은 제거한다.
- 시공관리 웹앱과 협력사 포털을 사용자 계정 + 권한 기반 구조로 전환한다.
- master 관리 기능은 시공관리 웹앱으로 통합한다.

## 1차 적용 항목

1. 모든 현장 리스트 기본 정렬을 시공일 내림차순으로 통일한다.
2. 시공일 없는 현장은 최하단에 배치한다.
3. 협력사 포털 상세 화면에 담당자 연락처를 기초데이터 시트 기준으로 연동한다.

## 다음 개발 묶음 후보

1. 시공관리 웹앱 로그인/회원가입/세션 8시간 유지
2. 기초데이터 기반 권한별 조회 범위 적용
3. 협력사데이터 신규 구조 반영 및 최초 로그인 비밀번호 변경
4. master 계정 승인/권한 변경/비밀번호 초기화 화면
5. NOTIFICATIONS 시트와 알림센터 구축

## 계정 DB 기준

### 시공관리 웹앱

- 시트: 기초데이터
- 로그인ID: 이름
- 비밀번호: 4자리 숫자
- 로그인 조건: 이름 일치, 비밀번호 일치, 사용여부 = 승인
- 권한: master, team1_leader, team2_leader, sales, viewer

### 협력사 포털

- 시트: 협력사데이터
- 로그인ID: 이름 + 휴대폰 뒤4자리
- 로그인 조건: 로그인ID 일치, 비밀번호 일치, 사용여부 = 승인
- 권한: partner, engineer

## 정렬 정책

- 적용 대상: 관리자, 팀장, 영업, viewer, partner, engineer
- 기준: 시공일 DESC
- 시공일 없음: 최하단

## 담당자 연락처 정책

- 담당자명은 현장 데이터의 담당자 값을 사용한다.
- 담당자 연락처는 기초데이터 시트의 이름(C열)과 휴대전화(E열)를 기준으로 매핑한다.
- 사용여부(H열)가 승인인 계정을 우선 사용한다.
