# 3번/4번 묶음 테스트 문서

최종 업데이트: 2026-05-28

## 3번. 파트너 포털 로그인 테스트

### 기준 시트

- 시트명: `협력사데이터`
- A열: 대리점명 / 협력사명
- B열: 이름 / 기사명
- C열: 연락처
- D열: 사업자번호
- E열: 비밀번호
- F열: 권한 `partner` 또는 `engineer`
- G열: 사용여부 `승인` 또는 `중지`
- H열: 비밀번호상태 `임시` 또는 `정상`

### 현재 코드 기준 로그인ID

- 최신 코드 기준 로그인ID는 B열 `이름 / 기사명`입니다.
- 과거 문서의 `이름 + 휴대폰 뒤4자리` 설명과 다르므로 실테스트는 B열 이름으로 진행합니다.

### 로그인 테스트 순서

1. `협력사데이터` 시트에 partner 계정 1개를 준비합니다.
2. F열 권한을 `partner`로 입력합니다.
3. G열 사용여부를 `승인`으로 입력합니다.
4. E열 비밀번호가 비어 있거나 `0000`이면 임시 비밀번호 로그인으로 테스트합니다.
5. 파트너 포털에서 B열 이름과 비밀번호로 로그인합니다.
6. 임시 비밀번호 상태라면 강제 비밀번호 변경 화면이 나와야 합니다.
7. 새 비밀번호는 숫자 4자리이며 `0000`은 사용할 수 없습니다.
8. 비밀번호 변경 후 E열은 `SHA256:` 해시값, H열은 `정상`으로 변경되어야 합니다.
9. 변경한 새 비밀번호로 다시 로그인합니다.

### 차단 테스트

1. G열 사용여부를 `중지`로 변경한 뒤 로그인합니다.
2. 중지된 계정 안내 문구가 표시되어야 합니다.
3. 비밀번호를 틀리게 입력하면 비밀번호 오류 문구가 표시되어야 합니다.
4. 존재하지 않는 이름으로 로그인하면 계정을 찾을 수 없다는 문구가 표시되어야 합니다.

## 4번. 협력사 현장 배정 흐름 테스트

### 사전 조건

1. 월 시트에 테스트 현장이 있어야 합니다.
2. U열 협력사가 partner 계정의 A열 대리점명과 정확히 같아야 합니다.
3. 같은 협력사에 engineer 계정이 1개 이상 있어야 합니다.
4. engineer 계정은 F열 권한 `engineer`, G열 사용여부 `승인`이어야 합니다.

### 현장 조회 테스트

1. partner 계정으로 파트너 포털에 로그인합니다.
2. 해당 협력사로 배정된 현장만 목록에 표시되는지 확인합니다.
3. 다른 협력사 현장이 표시되지 않아야 합니다.
4. 지급시공비는 partner 권한에서만 표시되어야 합니다.

### 엔지니어 배정 테스트

1. partner 계정으로 현장 상세 모달을 엽니다.
2. `엔지니어 배정` 영역에서 같은 협력사의 engineer를 선택합니다.
3. `배정` 버튼을 누릅니다.
4. 시트 V열에 엔지니어 이름이 저장되어야 합니다.
5. 시트 W열에 엔지니어 연락처가 저장되어야 합니다.
6. 시트 S열 상태가 `엔지니어배정완료`로 변경되어야 합니다.
7. 화면에서도 현장 카드/상세 모달의 엔지니어 정보가 갱신되어야 합니다.

### 권한/잠금 테스트

1. 다른 협력사 현장을 배정하려고 하면 `해당 협력사 현장이 아닙니다.` 메시지가 나와야 합니다.
2. 관리자 잠금 현장은 엔지니어 배정이 차단되어야 합니다.
3. engineer 권한 로그인에서는 엔지니어 배정 영역이 보이지 않아야 합니다.

## 관련 API

- 로그인: `partnerLogin`
- 비밀번호 변경: `partnerChangePassword`
- 현장 조회: `getPartnerJobs`
- 엔지니어 목록: `partnerInstallerData`
- 엔지니어 배정: `assignEngineer`
